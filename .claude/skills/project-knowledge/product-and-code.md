# DocTrack v2 — продукт, архитектура, карта папок

## Что делает проект
DocTrack — SaaS-система автоматизации документооборота для малого и среднего
бизнеса. Задача продукта — сократить ручные операции на 90% и повысить скорость
подписания документов до 10 минут. Продукт адресован всем компаниям с активным
документооборотом, не привязан к конкретной отрасли.

## Стек
- React 18 + TypeScript + Vite
- shadcn/ui (style: new-york, baseColor: neutral, cssVariables: true)
- Tailwind CSS
- No react-router — routing via `activePage` state in `App.tsx`

## Aliases
- @/components → src/components
- @/components/ui → src/components/ui
- @/lib → src/lib
- @/hooks → src/hooks

## Где что лежит
- `src/mocks` — моковые данные
- `src/types` — TypeScript-типы
- `src/constants` — константы, справочники
- `src/components` — компоненты
  - `src/components/ui` — shadcn-примитивы
  - `src/components/base` — базовые переиспользуемые компоненты проекта
- `src/hooks` — хуки
- `src/lib` — утилиты, бизнес-логика
- `src/pages` — страницы
  - `src/pages/base` — страницы раздела БАЗА
- `docs/clients-reference.tsx` — полный UI-референс страницы Клиенты. Читать
  перед созданием любых list/table view.

## Где искать существующее перед тем, как писать новое
Перед созданием нового хука или утилиты — проверь `src/hooks` и `src/lib`.
Перед добавлением моковых данных — проверь `src/mocks`, не создан ли мок
для этой сущности уже.
