# Часть A — как заполнять

## Шаг 0 — вход

Прочитай **только**:
- `prompts/NN-название.md` — базовый промпт по этой фиче;
- `prompts/NN-название.report.md` — отчёт по последней выполненной итерации
  (если итераций уже несколько — только самый свежий, не всю историю).

Зафиксированные там допущения (произвольные ФИО, конкретная реализация,
выбранная "на усмотрение", и т.п.) — не переигрываются, если пользователь
явно не просит их пересмотреть. Остальные файлы project-knowledge читать
только если в этой сессии их ещё не читал (см. закрытый список в SKILL.md).

## Шаг 1 — заполнение разделов

1. Скопируй разметку из Части B ниже в `prompts/NN-название.delta-K.md`.
2. Раздел 2b дельты — та же механика (grep импортёров), но только по
   символам, которые трогает эта правка. Раздел 2a не повторяется —
   действует 2a базового промпта, ссылкой.
3. Раздел 3 — только новое/изменённое:
   - новые требования продолжают нумерацию базового промпта (был R11 →
     новое R12);
   - изменённые получают суффикс `R6 → R6.1`, с явной пометкой "заменяет
     R6, старая формулировка недействительна";
   - номера не переиспользуются никогда.
4. Раздел 3a (только если правка не тривиальна — больше одного нового R
   или есть регрессия) — трассировка новых R → новых/затронутых D.
5. Раздел 4 — новые D под новые R, плюс отдельный список "регрессия":
   D-id из базового промпта, которые могла задеть эта правка. Определяется
   по пересечению файлов/символов, затронутых правкой, с файлами/символами,
   на которые ссылаются D-id базового промпта — не интуитивно.
6. Раздел 5 — шаги только под новые и регрессионные D-id, не весь набор
   заново.
7. Раздел 6 — обязательный шаг: сверить, что допущения из базового отчёта
   не нарушены этой правкой; новая секция в том же файле отчёта, не новый
   файл.

## Самопроверка перед показом пользователю

Та же таблица, что в `template.md`, плюс:

| Пункт | Где править | Что перепроверить |
|---|---|---|
| номер R/D переиспользован вместо суффикса | нумерация в разделе 3/4 | 3a, регрессия |
| регрессия не проверена по факту пересечения файлов | список регрессии в разделе 4 | шаги раздела 5 |
| допущение базового отчёта переиграно без просьбы пользователя | раздел 3/6 | — |
