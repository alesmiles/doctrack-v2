## 1) Заголовок

Перевести экран редактора документа на русский язык и наполнить тело документа данными
из мок-сметы, привязанной к проекту. Цель: пользователь открывает редактор после загрузки
сметы и видит готовый русскоязычный документ с реальными данными (услуги, стоимость, сроки).

## 3) Описание задачи

**R1.** Весь UI редактора документа (`src/pages/DocumentEditorPage.tsx`) переведён на
русский: топбар («← Назад», «Предпросмотр», «Экспорт .docx», «Доступно после устранения
всех замечаний»), панель замечаний («Замечания», «N требуют внимания» / «Всё готово»,
«Прогресс»), диалог предпросмотра («Предпросмотр документа», «Закрыть»), тост при
экспорте («Документ экспортирован»).

**R2.** Четыре замечания в `DocumentEditorPage.tsx` (массив `ISSUES`) переведены на
русский: заголовки, описания и формулировки-подсказки (`formulation`).

**R3.** Тело документа (`src/components/DocumentBody.tsx`) переведено на русский: шапка
реквизитов, преамбула сторон, заголовки разделов («1. ПРЕДМЕТ ДОГОВОРА», «2. СРОКИ»,
«3. ПРИЁМКА», «4. СТОИМОСТЬ»), текст пунктов.

**R4.** Создан файл `src/mocks/smetaMock.ts` с мок-данными сметы для проекта `"ALF-9 · March 2025"`:
- услуги: массив `{ name: string; qty: number; unit: string; price: number }` — минимум 3 позиции
  (например: «Размещение у блогеров», «Создание контента», «Аналитика и отчётность»)
- `totalAmount: number` — итоговая сумма (394 000 руб.)
- `period: string` — период выполнения («март — май 2025»)
- `projectCode: string` — `"ALF-9"`

**R5.** `DocumentBody` принимает проп `smetaData?: SmetaData` (тип из `smetaMock.ts`).
Когда `smetaData` передан — раздел «1. ПРЕДМЕТ» перечисляет услуги из `smetaData.services`
(название, кол-во, ед., цена); раздел «2. СРОКИ» использует `smetaData.period`; раздел
«4. СТОИМОСТЬ» использует `smetaData.totalAmount`. Когда `smetaData` не передан — документ
рендерится с плейсхолдерами (текущее поведение сохраняется).

**R6.** `App.tsx` сохраняет форму создания документа в state `savedClientDocForm` (уже
существует). При переходе на `"document-editor"` через `onNavigate` контекст
`{ project: string }` передаётся в `DocumentEditorPage` через новый проп `docContext`.
`DocumentEditorPage` по `docContext.project` ищет мок сметы в `smetaMock.ts` и передаёт
найденный объект в `DocumentBody` через `smetaData`.

## 4) Definition of Done

**D1.** Кнопка «назад» в топбаре содержит текст «← Назад» (не «← Back»). [проверяет: R1]

**D2.** Кнопка предпросмотра содержит текст «Предпросмотр» (не «Preview»). [проверяет: R1]

**D3.** Кнопка экспорта содержит текст «Экспорт .docx» (не «Export .docx»). [проверяет: R1]

**D4.** Подпись под заблокированной кнопкой экспорта содержит текст «Доступно после
устранения всех замечаний» (не «Available after all issues are resolved»). [проверяет: R1]

**D5.** Заголовок боковой панели содержит «Замечания», счётчик — «N требуют внимания»
(не «Issues», не «N need attention»); метка «Прогресс» отображается в подвале панели. [проверяет: R1]

**D6.** Первое замечание в списке имеет заголовок на русском языке (не «Ad Format
Breakdown Required»). [проверяет: R2]

**D7.** Заголовок первого раздела в теле документа содержит «ПРЕДМЕТ» (не «SCOPE OF WORK»). [проверяет: R3]

**D8.** Заголовок четвёртого раздела содержит «СТОИМОСТЬ» (не «FEES»). [проверяет: R3]

**D9.** Файл `src/mocks/smetaMock.ts` существует и экспортирует объект с полями
`services` (массив), `totalAmount` (число), `period` (строка). [проверяет: R4]

**D10.** В теле документа раздел «1. ПРЕДМЕТ» содержит перечисление услуг из мок-сметы
(не менее 3 строк с названиями услуг). [проверяет: R5]

**D11.** Раздел «4. СТОИМОСТЬ» содержит строку «394 000» в тексте документа. [проверяет: R5]

**D12.** При открытии редактора документа через попап «Документ клиента» с проектом
«ALF-9 · March 2025» тело документа содержит данные из мок-сметы (проверяется D10 и D11
через этот сценарий). [проверяет: R6]

## 5) Проверка через Playwright (MCP, реальный браузер)

Проверка идёт через инструменты Playwright MCP по живому dev-серверу, не через
testing-library/jest/vitest (их в проекте нет).

1. Открыть `http://localhost:5173` → снапшот → убедиться, что приложение загрузилось. [проверяет: D12]

2. Нажать «Создать» в сайдбаре → «Документ клиента» → снапшот → убедиться, что открылся
   модал «Документ клиента». [проверяет: D12]

3. Выбрать клиента «Alpha Media», проект «ALF-9 · March 2025», тип «Приложение», дату
   подписания любую, МенДО любой → загрузить любой файл через зону загрузки → дождаться
   состояния «ready» → снапшот → убедиться, что появилась кнопка «Открыть документ». [проверяет: D12]

4. Нажать «Создать документ» → снапшот → убедиться, что открылся редактор документа. [проверяет: D1]

5. Снапшот топбара → проверить наличие текста «← Назад», «Предпросмотр», «Экспорт .docx». [проверяет: D1] [проверяет: D2] [проверяет: D3]

6. Снапшот боковой панели → проверить наличие «Замечания», «требуют внимания», первого
   замечания на русском, «Прогресс». [проверяет: D5] [проверяет: D6]

7. Снапшот тела документа → проверить наличие «ПРЕДМЕТ» и «СТОИМОСТЬ» в заголовках
   разделов. [проверяет: D7] [проверяет: D8]

8. Снапшот тела документа → проверить, что раздел «1. ПРЕДМЕТ» содержит хотя бы 3
   строки с названиями услуг. [проверяет: D9] [проверяет: D10]

9. Снапшот тела документа → проверить наличие «394 000» в разделе стоимости. [проверяет: D11]

10. Нажать «Предпросмотр» → снапшот → проверить наличие текста «Предпросмотр документа»
    и кнопки «Закрыть». [проверяет: D2]

11. Закрыть предпросмотр → нажать на первое замечание → принять подсказку → снапшот →
    убедиться, что замечание помечено решённым. [проверяет: D6]

12. Решить все 4 замечания → снапшот → убедиться, что кнопка «Экспорт .docx» стала
    активной и подпись «Доступно после устранения всех замечаний» исчезла. [проверяет: D3] [проверяет: D4]

13. Нажать «Экспорт .docx» → снапшот → убедиться, что появился тост «Документ
    экспортирован». [проверяет: D3]

14. Открыть консоль браузера → проверить на 0 ошибок. [проверяет: D12]
