# DocTrack v2 — Claude Code Instructions

## Проект
DocTrack — SaaS автоматизации документооборота для компаний с активным
документооборотом (не привязан к отрасли). Цель: сократить ручные операции
на 90%, подписание до 10 минут.

## Запуск и проверки
- `npm install && npm run dev` — vite, http://localhost:5173
- `npm run lint` / `npm run typecheck` / `npm run build` — все три обязаны
  проходить перед сдачей любой задачи
- Автотестов пока нет — появятся позже. Функциональная проверка сейчас —
  через Playwright (MCP) по сценарию задачи.
- Supabase в зависимостях, пока не используется. Деплоя пока нет — проект
  работает локально.

## База знаний
Перед началом задачи открой `.claude/skills/project-knowledge/SKILL.md`
и прочитай только файлы, указанные там для твоего типа задачи. Не читай всё подряд.

## Нельзя без согласования
- `git commit`, `git push`, деплой
- Удалять файлы и данные
Это ограничение на действия, не на код: редактировать код в рамках задачи можно свободно.

## Режим работы (обязательно для всех задач)
ЗАПРЕЩЕНО задавать уточняющие вопросы или останавливаться в ожидании ответа
в чате на любом этапе выполнения задачи. При неоднозначности — выбери наиболее
вероятную трактовку, реализуй, опиши допущение только в финальном отчёте.
Работай непрерывно до конца чек-листа без остановок на подтверждение в чате.
Исключение — системные диалоги macOS/терминала (разрешение на запись файлов,
bash-команды) — они не текстовые вопросы, на них запрет не действует.

(Эталон этого текста — .claude/skills/doctrack-prompt-builder/reference/
section7_canonical.txt. При изменении править эталон и синхронизировать сюда.)

## Пополнение BASE

При добавлении нового клиента в `src/mocks/baseClients.ts` или подрядчика в
`src/mocks/baseContractors.ts` он **автоматически** появляется в фильтрах и
дропдаунах всех модальных окон (через `CLIENT_NAMES` / `CONTRACTOR_NAMES`
из `src/utils/clients.ts` / `src/utils/contractors.ts`).

Чтобы компания появилась в **канбан-карточках** и списке проектов — добавить
запись вручную:
- Клиент → `src/mocks/projects.ts` (поле `client` должно точно совпадать с
  `company` в BASE_CLIENTS, включая регистр)
- Подрядчик → `src/mocks/contractorsKanbanMockData.ts` (поле `contractorName`)
