## 3) Описание правки

**Новые:**

**R7.** `DocumentBody.tsx` дополнен разделами, которых нет в текущей версии:
«5. ПОРЯДОК РАСЧЁТОВ» (предоплата/постоплата, срок оплаты счёта),
«6. ОТВЕТСТВЕННОСТЬ СТОРОН» (пени за просрочку оплаты и за просрочку услуг),
«7. ПРОЧИЕ УСЛОВИЯ» (срок действия, количество экземпляров, применимое право),
«8. РЕКВИЗИТЫ И ПОДПИСИ СТОРОН» (блок с ИНН, юр. адресом, банковскими реквизитами,
строками «Подпись: ______», «Ф.И.О.: ______», «Печать» для каждой из сторон).
Данные Исполнителя берутся из захардкоженного мока (ООО «Агентство»), данные Заказчика —
из `smetaData` (клиент) либо плейсхолдер когда `smetaData` не передан.

**R8.** В развёрнутой карточке замечания (`IssueCard.tsx`) добавлена кнопка «Не менять»
между «Принять» и «Изменить». Нажатие закрывает замечание и добавляет его в `resolvedIds`,
но **не** вносит `formulation` в документ — исходный текст метки остаётся без изменений.
Расположение кнопок: первая строка `[Принять] [Не менять] [Изменить]`, вторая строка —
текстовая ссылка «Отклонить» (без рамки, мелкий текст, не меняет смысл: закрыть панель
без решения). Для реализации «Не менять» добавляется проп `onKeepOriginal: (id: string) =>
void` в `IssueCard`; в `DocumentEditorPage` он добавляет `id` только в `resolvedIds`,
не трогая `resolvedTexts`.

**R9.** На свёрнутой карточке замечания при наведении (hover) справа появляются две
иконки-кнопки, видимые только пока курсор над карточкой: ✓ (принять предложенное — то же
что «Принять» в развёрнутом виде) и — (оставить без изменений — то же что «Не менять»).
Клик на иконку срабатывает без раскрытия карточки. Иконки не отображаются, если замечание
уже решено (`isResolved`).

**R10.** Кнопка «Экспорт .docx» в `DocumentEditorPage.tsx` при клике (когда `allDone`)
выполняет реальное скачивание `.docx`-файла. Реализация: установить npm-пакет
`html-docx-js` (или `html-docx-js-typescript`); взять `innerHTML`
контейнера `DocumentBody` из DOM через `ref`; обернуть в минимальную HTML-обёртку
(`<html><body>…</body></html>`); передать в `htmlDocx.asBlob()`; создать
`URL.createObjectURL` и программно кликнуть `<a download="приложение-ALF-9-03.docx">`.
Тост «Документ экспортирован» сохраняется. Если пакет `html-docx-js` не устанавливается —
использовать альтернативу `html-to-docx` с той же схемой.

## 4) Definition of Done (новое + регрессия)

**Новые:**

**D13.** Тело документа содержит заголовок «5. ПОРЯДОК РАСЧЁТОВ» и заголовок
«7. ПРОЧИЕ УСЛОВИЯ». [проверяет: R7]

**D14.** Тело документа содержит блок «8. РЕКВИЗИТЫ И ПОДПИСИ СТОРОН» со строками
«Подпись:» и «Ф.И.О.:» для каждой из двух сторон. [проверяет: R7]

**D15.** В развёрнутой карточке замечания присутствует кнопка с текстом «Не менять»
между «Принять» и «Изменить». [проверяет: R8]

**D16.** После нажатия «Не менять» замечание переходит в состояние resolved (зачёркнуто,
счётчик прогресса уменьшается), а соответствующая метка в теле документа отображает
исходный текст (не `formulation`). [проверяет: R8]

**D17.** При наведении на свёрнутую нерешённую карточку замечания появляются две иконки
справа от заголовка. [проверяет: R9]

**D18.** Клик на иконку ✓ на свёрнутой карточке разрешает замечание с текстом
`formulation`, карточка не раскрывается. [проверяет: R9]

**D19.** Клик на иконку — на свёрнутой карточке разрешает замечание без изменения
исходного текста, карточка не раскрывается. [проверяет: R9]

**D20.** При клике «Экспорт .docx» (все замечания решены) браузер скачивает файл с
расширением `.docx` (не `.txt`, не `.html`). [проверяет: R10]

**D21.** Скачанный файл открывается в Microsoft Word и содержит текст «ПРЕДМЕТ
ДОГОВОРА» в теле. [проверяет: R10]

**Регрессия (не нумеруются — D-id из базового промпта):**

- Кнопка «Экспорт .docx» затронута R10: текст кнопки должен остаться «Экспорт .docx»,
  кнопка по-прежнему неактивна до решения всех замечаний.
- `IssueCard` затронут R8 и R9: первое замечание по-прежнему отображается с заголовком
  на русском языке.
- `DocumentBody` затронут R7: раздел «1. ПРЕДМЕТ» по-прежнему содержит перечисление
  услуг из мок-сметы.

## 5) Проверка (новое + регрессионные шаги)

Проверка идёт через Playwright MCP по живому dev-серверу, не через
testing-library/jest/vitest.

1. Открыть редактор документа (Создать → Документ клиента → ALF-9 → Создать документ) →
   снапшот тела документа → убедиться, что присутствуют «5. ПОРЯДОК РАСЧЁТОВ» и
   «7. ПРОЧИЕ УСЛОВИЯ». [проверяет: D13]

2. Снапшот нижней части документа → убедиться, что блок «8. РЕКВИЗИТЫ И ПОДПИСИ СТОРОН»
   содержит строку «Подпись:». [проверяет: D14]

3. Снапшот тела документа → убедиться, что раздел «1. ПРЕДМЕТ» содержит услуги из сметы
   (регрессия). [проверяет: D13]

4. Раскрыть первое замечание → снапшот → убедиться, что кнопка «Не менять» присутствует
   между «Принять» и «Изменить». [проверяет: D15]

5. Убедиться, что первое замечание имеет заголовок на русском (регрессия). [проверяет: D15]

6. Нажать «Не менять» на первом замечании → снапшот боковой панели → убедиться, что
   замечание зачёркнуто, прогресс показывает «1 of 4». [проверяет: D16]

7. Снапшот тела документа → убедиться, что метка первого замечания показывает исходный
   текст (не `formulation`). [проверяет: D16]

8. Навести курсор на второе нерешённое замечание (свёрнутое) → снапшот → убедиться, что
   появились две иконки справа. [проверяет: D17]

9. Кликнуть иконку ✓ на свёрнутом замечании → снапшот → убедиться, что замечание
   решено, карточка не раскрывалась (описание замечания не видно). [проверяет: D18]

10. Кликнуть иконку — на третьем нерешённом замечании → снапшот → убедиться, что решено
    без изменения исходного текста метки в документе. [проверяет: D19]

11. Решить последнее замечание любым способом → снапшот топбара → убедиться, что кнопка
    «Экспорт .docx» активна и подпись «Доступно после...» исчезла (регрессия). [проверяет: D20]

12. Нажать «Экспорт .docx» → проверить, что в папке загрузок появился файл с расширением
    `.docx`. [проверяет: D20]

13. Открыть скачанный файл → убедиться, что содержит текст «ПРЕДМЕТ ДОГОВОРА». [проверяет: D21]

14. Проверить консоль браузера на 0 ошибок. [проверяет: D20]
