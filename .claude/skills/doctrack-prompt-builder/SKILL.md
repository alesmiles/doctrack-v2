# DocTrack — сборка промптов

## Режимы

1. **Новая фича** — задачи в проекте ещё не было → `reference/template.md`,
   Часть A целиком (шаг 0 — выбор пути, шаг 1 — заполнение разделов,
   самопроверка перед показом).
2. **Доработка после итерации** — в `prompts/` уже есть промпт и
   `*.report.md` по этой же фиче → `reference/template-delta.md`.
3. **Проверка присланного промпта** — пользователь прислал готовый текст,
   не просит сгенерировать новый → см. «Проверка присланного промпта» ниже.

Перед началом ВСЕГДА проверь `prompts/` на промпт и отчёт по этой же фиче
(по имени файла / grep компонента-сущности из задачи). Нашёл → режим 2, не
режим 1, без явной просьбы пользователя на полную перегенерацию.

## Закрытый список чтения перед стартом

Ровно это, не больше — «на всякий случай» не читать:

1. `product-and-code.md` + `style-and-patterns.md` (project-knowledge skill),
   если ещё не читались в этой сессии.
2. `docs/clients-reference.tsx` — только если задача про list/table/kanban view.
3. Для режима 2 (доработка) — только `prompts/NN-название.md` и
   `prompts/NN-название.report.md` по этой же фиче, не остальные файлы
   в `prompts/`.
4. Ссылки на конкретный код (файл/строка/символ) в разделах 2a/2b/3 —
   проверяются в реальном файле непосредственно перед тем, как попасть в
   промпт, не восстанавливаются по памяти о прошлой сессии. Это не
   отдельный проход «прочитать всё заново для проверки» — файл, прочитанный
   минуту назад в рамках этого же запуска, повторно не перечитывается.

## Линт (механика, отдельно от смысловой самопроверки)

При сохранении файла в `prompts/` линт запускается автоматически через хук
(`.claude/settings.json`), результат приходит в контекст без отдельного
вызова. Если хук не сработал — прогони `scripts/lint_prompt.py` вручную до
показа промпта пользователю.

Линт ловит только механику: пропущенные/пустые разделы, запрещённые
формулировки без ссылки на файл (`reference/banned_phrases.txt`),
расхождение раздела 7 с эталоном, обязательные элементы раздела 6,
трассировку R↔D, покрытие D шагами раздела 5, ссылки на несуществующий код,
конфликт «или»/скобочных вариантов в разделе 4, тестовые матчеры
(Testing Library) в разделах 4 и 5.

Прохождение линта не равно готовности — таблица самопроверки в конце
Части A (`template.md` / `template-delta.md`) обязательна в любом случае.

При провале пункта самопроверки — исправь точечно этот раздел по колонке
«Где править / Что перепроверить» в этой же таблице, не весь процесс с
нуля. Полный передел — только если на шаге 0 (template.md) выбран не тот
путь (лёгкий/полный).

## Проверка присланного промпта

Тот же процесс — линт + таблица самопроверки — применяется к присланному
тексту напрямую. Результат — список конкретных несоответствий по пунктам
таблицы, с указанием, что и где нарушено. Не переписанный промпт.

## Файлы этого skill

- `reference/template.md` — инструкция + скелет для новой фичи. Часть A не
  копируется в `prompts/`, только читается как процесс. Часть B копируется
  целиком в `prompts/NN-название.md`.
- `reference/template-delta.md` — то же для доработки после итерации,
  результат — `prompts/NN-название.delta-K.md`.
- `reference/section7_canonical.txt` — эталонный текст раздела 7,
  вставляется дословно, не переформулируется и не пересказывается.
- `reference/banned_phrases.txt` — запрещённые оценочные формулировки.
- `scripts/lint_prompt.py` — механический линтер.

## Важно

Изменение самих `template.md` / `template-delta.md` — отдельное действие,
не выполняется молча внутри задачи по генерации промпта.

## Граница задачи — стоп после показа промпта

После показа Части B пользователю — остановиться полностью.

ЗАПРЕЩЕНО в рамках этой задачи:
- Запускать Playwright, браузер, dev-сервер.
- Выполнять шаги раздела 5 из только что написанного промпта.
- Проверять UI, кликать, делать скриншоты.

Раздел 5 — инструкция для сессии-исполнителя, которая будет реализовывать
фичу. Prompt-builder её не выполняет.

## Лог эффективности

После каждого запуска дописать одну строку в `prompts/prompt-builder.log.md`
(создать если нет):

```
| дата | режим | время | токены | читал сверх списка | линт-циклов | лишние действия | качество |
```

- **читал сверх списка** — любой файл не из закрытого списка чтения выше.
- **линт-циклов** — сколько раз пришлось править промпт после линта.
- **лишние действия** — Playwright, повторное чтение уже прочитанных файлов,
  grep сверх алгоритма 2b.
- **качество** — пришлось ли пользователю вручную доделывать промпт после
  показа (да/нет, коротко что).

Лог не показывается пользователю — пишется молча как последний шаг.
